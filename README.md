## 🚀 Getting Started

### Prerequisites

```
✓ Java Development Kit (JDK) 8+
✓ Gradle build system
✓ Git version control
✓ Minecraft Java Edition
✓ Your favorite code editor
```

### ⚡ Quick Setup

```bash
# 1️⃣ Clone the repository
git clone https://github.com/Blessonsharon/Minecraft-Modding.git
cd Minecraft-Modding

# 2️⃣ Build the project
gradle build

# 3️⃣ Run tests
gradle test

# 4️⃣ Generate Javadoc
gradle javadoc
```

## 📥 Installation

### 👤 For Players

1. 📥 Download from [**Releases**](https://github.com/Blessonsharon/Minecraft-Modding/releases)
2. 📂 Navigate to your Minecraft mods folder:
   ```
   Windows:  %APPDATA%\.minecraft\mods
   macOS:    ~/Library/Application Support/minecraft/mods
   Linux:    ~/.minecraft/mods
   ```
3. 📋 Place the `.jar` file in the mods folder
4. 🎮 Launch Minecraft with your mod loader (Forge, Fabric, etc.)
5. ✨ Enjoy your enhanced gameplay!

### 👨‍💻 For Developers

```bash
# Fork the repository on GitHub
# Clone your fork locally
git clone https://github.com/YOUR-USERNAME/Minecraft-Modding.git

# Create a feature branch
git checkout -b feature/YourFeature

# Start developing!
```

## 🤝 Contributing

### 🎯 Contribution Workflow

```
1. 🍴 Fork the repository
   └─ Click "Fork" on GitHub
   
2. 🌿 Create feature branch
   └─ git checkout -b feature/amazing-feature
   
3. ✏️ Make your changes
   └─ Write clean, documented code
   
4. ✅ Commit changes
   └─ git commit -m "Add amazing feature"
   
5. 📤 Push to branch
   └─ git push origin feature/amazing-feature
   
6. 🔄 Open Pull Request
   └─ Describe changes in detail
```

### 📋 Contributing Guidelines

- ✅ Follow Java naming conventions
- ✅ Add comments for complex logic
- ✅ Test your changes thoroughly
- ✅ Update documentation
- ✅ Keep commits atomic and descriptive
- ✅ One feature per pull request
